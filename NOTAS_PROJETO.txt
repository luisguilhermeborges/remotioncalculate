COMO FUNCIONA A HIERARQUIA DE PERMISSÕES
------------------------------------------------------------
1. Admin (login admin@admin.com) → acesso total
2. Membros com role "Gerente", "Night Boss", "Owner" → acesso total
3. Outros membros → acesso baseado nas permissões do cargo (Roles)

Para Área Ilegal:
- Membro precisa ter flagIlegal = true
- Cargo ilegal separado do cargo da mecânica
- Membros "Agregado" = só aparecem na área ilegal, escondidos da mecânica

------------------------------------------------------------
NIVEIS DE AUTORIZAÇÃO (Cargos)
------------------------------------------------------------
Mecânica:
  - Night Boss / CEO / Gerente → acesso completo
  - Racer / Mecanico / Estagiario → acesso limitado (configurável)
  - Runner → acesso básico

Ilegal (flagIlegal = true):
  - Leader → topo
  - Sub-Leader
  - Soldado
  - Novato

Agregado = membro somente do ilegal (não aparece na hierarquia da mecânica)

CONFIGURAÇÃO DO SUPABASE (opcional)
------------------------------------------------------------
Se tiver banco online:
1. Clicar no badge de status (canto superior esquerdo do conteúdo)
2. Inserir URL e Anon Key do Supabase
3. Salvar — a página recarrega e usa o banco online

Sem Supabase → todos os dados ficam no localStorage do navegador.

Tabelas necessárias no Supabase:
  - materials (id, name, price, image)
  - items (id, category, name, stage, price, sell_price, ingredients, image)
  - members (id, name, role, status, passport, phone, joinDate, password, flagIlegal, illegalRole)
  - mural (id, created_at, title, content, author, tag)
  - vault_logs (id, timestamp, member, action, item, quantity)
  - illegal_records (id, timestamp, description, outcome, participants, loot)
  - impounded_cars (id, model, plate, owner, retrieved)
  - illegal_recipes (id, name, time, image, ingredients, recipeId)
